<h1>NeoCraft — Integration Guide (Minecraft 1.12.2)</h1><p>This guide walks you through wiring NeoCraft into an <strong>Eaglercraft 1.12.2 workspace</strong> (or any MCP-mapped Minecraft 1.12.2 development environment). NeoCraft's core is fully portable and contains no proprietary code; the bridge in <code>/integration</code> is the only piece that touches Minecraft classes, and it is designed to be completed inside your workspace.</p><hr><h2>Table of Contents</h2><ul> <li><a href="#prerequisites">Prerequisites</a></li> <li><a href="#overview">Overview</a></li> <li><a href="#step-1--add-the-jars-to-your-workspace">Step 1 — Add the jars to your workspace</a></li> <li><a href="#step-2--complete-the-mcp-bridge">Step 2 — Complete the MCP bridge</a></li> <li><a href="#step-3--initialize-neocraft">Step 3 — Initialize NeoCraft</a></li> <li><a href="#step-4--route-game-events">Step 4 — Route game events</a></li> <li><a href="#step-5--open-the-guis">Step 5 — Open the GUIs</a></li> <li><a href="#step-6--wire-the-cps-counter">Step 6 — Wire the CPS counter</a></li> <li><a href="#step-7--verify">Step 7 — Verify</a></li> <li><a href="#adapter-method-reference">Adapter method reference</a></li> </ul><hr><h2>Prerequisites</h2><ul> <li>A working <strong>Eaglercraft 1.12.2</strong> development workspace (or a Forge/ForgeGradle MCP-mapped 1.12.2 environment) with <code>net.minecraft.*</code> classes on the classpath.</li> <li><strong>JDK 17</strong> for building the NeoCraft jars.</li> <li>The NeoCraft core jar (<code>NeoCraft-1.0.0-core.jar</code>) and the integration jar (<code>NeoCraft-1.0.0-integration.jar</code>), built via <code>gradle build</code> and <code>gradle integrationJar</code>.</li> </ul><hr><h2>Overview</h2><p>NeoCraft uses an adapter pattern. The core (<code>org.neocraft.*</code>) only ever calls methods on four interfaces:</p><ul> <li><code>MinecraftAdapter</code> — input, timing, player/world state, rendering entry points, config.</li> <li><code>GameRenderer</code> — 2D immediate-mode drawing (rects, gradients, circles, blur, particles, scissor, blend).</li> <li><code>FontRenderer</code> — string drawing, measurement, wrapping.</li> <li><code>ConfigStore</code> — key/value persistence with namespaces.</li> </ul><p>The <code>integration</code> source set provides <code>McpAdapter</code>, <code>McpRenderer</code>, <code>McpFont</code>, and <code>McpConfig</code> — concrete implementations of these interfaces. They ship as <strong>commented-out reference implementations</strong> that throw <code>UnsupportedOperationException</code> so they compile in the sandbox. Your job is to uncomment the real MCP calls.</p><pre><code>Your mod  ──▶  NeoCraft.init(new McpAdapter())
                       │
            McpAdapter ─┬─▶ McpRenderer  (Tessellator / GlStateManager)
                        ├─▶ McpFont      (mc.fontRenderer)
                        └─▶ McpConfig    (Properties file on disk)
</code></pre><hr><h2>Step 1 — Add the jars to your workspace</h2><p><strong>Option A — Jars on the classpath:</strong></p><p>Copy both jars into your workspace's <code>mods/</code> or <code>libs/</code> directory and add them to your build's <code>compileOnly</code>/<code>implementation</code> dependencies:</p><pre><code class="language-groovy">dependencies {
    implementation files('libs/NeoCraft-1.0.0-core.jar')
    implementation files('libs/NeoCraft-1.0.0-integration.jar')
}
</code></pre><p><strong>Option B — Source files (recommended for editing the bridge):</strong></p><p>Copy the entire <code>integration/org/neocraft/adapter/</code> tree and the <code>src/main/java/org/neocraft/</code> tree into your workspace's source directory. This gives you direct access to uncomment and edit the MCP calls.</p><p>Either way, ensure the <code>org.neocraft.api</code> interfaces and <code>org.neocraft.adapter.Mcp*</code> classes resolve.</p><hr><h2>Step 2 — Complete the MCP bridge</h2><p>Open each file in <code>integration/org/neocraft/adapter/</code> and uncomment the implementation hints. The comments describe the Tessellator/GlStateManager approach for each primitive. For <code>drawRect</code>, the vanilla <code>Gui.drawRect</code> is a good starting point; for <code>drawRoundedRect</code>, compose corner quarter-circles plus two rectangles; for <code>drawBlur</code>, use a stencil-based Gaussian blur shader pass. <code>enableScissor</code>/<code>disableScissor</code> map to <code>GlStateManager.enableScissor()</code>/<code>disableScissor()</code> with display-resolution scaling.</p><h3>McpFont.java</h3><p>Uncomment the constructor (<code>this.fr = mc.fontRenderer;</code>) and delegate every method to the vanilla <code>FontRenderer</code>. The ARGB int format NeoCraft uses is directly compatible with <code>fr.drawString(text, x, y, argb, shadow)</code>.</p><h3>McpConfig.java</h3><p>Uncomment the <code>Properties</code>-backed implementation. The default constructor points at <code>new File(Minecraft.getMinecraft().gameDir, "neocraft/config.properties")</code>. Namespaces use a key-prefix layering scheme (<code>profile.PvP.&lt;key&gt;</code>), and child stores share the underlying <code>Properties</code> object with their parent for atomic saves.</p><hr><h2>Step 3 — Initialize NeoCraft</h2><p>From your mod's init event handler (Eaglercraft's <code>ModInitializer</code>, Forge's <code>FMLInitializationEvent</code>, or your bootstrap equivalent), call:</p><pre><code class="language-java">import org.neocraft.client.NeoCraft;
import org.neocraft.adapter.McpAdapter;

@EventHandler
public void init(FMLInitializationEvent event) {
    NeoCraft.init(new McpAdapter());
}
</code></pre><p><code>NeoCraft.init()</code> is idempotent and safe to call once. It will:</p><ol> <li>Load the persisted theme selection.</li> <li>Register all built-in modules and load their persisted state.</li> <li>Load the HUD layout.</li> <li>Restore the last active profile.</li> <li>Load the resource pack toggle.</li> <li>Print the NeoCraft startup banner.</li> </ol><p>On shutdown, call <code>NeoCraft.get().shutdown()</code> to persist all state.</p><hr><h2>Step 4 — Route game events</h2><p>NeoCraft's <code>EventBus</code> expects three event types from the host:</p><h3>Tick event (every game tick)</h3><pre><code class="language-java">NeoCraft.get().events().post(new org.neocraft.events.TickEvent());
</code></pre><p>Fire this from your tick handler (e.g. <code>ClientTickEvent.Phase.END</code>). Modules like Sprint and Fullbright react here.</p><h3>Render 2D event (every frame, after HUD)</h3><pre><code class="language-java">NeoCraft.get().events().post(new org.neocraft.events.Render2DEvent(partialTicks));
</code></pre><p>Fire this from your <code>RenderGameOverlayEvent.Post</code> handler (or Eaglercraft's overlay render hook). The HUD manager and notifications render here.</p><h3>Key input event (on key press)</h3><pre><code class="language-java">NeoCraft.get().events().post(new org.neocraft.events.KeyInputEvent(keyCode));
</code></pre><p>Fire this from your key input handler. The module manager checks keybinds and toggles modules here.</p><p>The event bus is annotation-driven: modules subscribe with <code>@NeoSubscribe</code> methods. You do not need to manually dispatch to individual modules — just post the three events above.</p><hr><h2>Step 5 — Open the GUIs</h2><p>To open the ClickGUI (typically bound to Right Shift or a configurable key):</p><pre><code class="language-java">import org.neocraft.gui.ClickGui;
import org.neocraft.client.NeoCraft;

// In your key handler or GUI-open hook:
ClickGui gui = new ClickGui(NeoCraft.get().adapter());
mc.displayGuiScreen(gui);  // or your Eaglercraft equivalent
</code></pre><p>To open the redesigned main menu, settings, or credits screens:</p><pre><code class="language-java">new org.neocraft.gui.MainMenu(NeoCraft.get().adapter());
new org.neocraft.gui.SettingsScreen(NeoCraft.get().adapter());
new org.neocraft.gui.CreditsScreen(NeoCraft.get().adapter());
</code></pre><p>The <code>MainMenu</code> exposes a <code>MenuAction</code> callback so you can wire its buttons to your workspace's existing singleplayer/multiplayer screens.</p><hr><h2>Step 6 — Wire the CPS counter</h2><p><code>getLeftClickCps()</code> and <code>getRightClickCps()</code> need a small CPS counter in <code>McpAdapter</code>. Add a timestamp deque and prune clicks older than one second:</p><pre><code class="language-java">private final java.util.ArrayDeque&lt;Long&gt; leftClicks = new java.util.ArrayDeque&lt;&gt;();
private final java.util.ArrayDeque&lt;Long&gt; rightClicks = new java.util.ArrayDeque&lt;&gt;();

// Call from your mouse-click handler (onClick / onMiddleClick etc.):
public void onLeftClick() {
    leftClicks.addLast(System.currentTimeMillis());
    prune(leftClicks);
}
public void onRightClick() {
    rightClicks.addLast(System.currentTimeMillis());
    prune(rightClicks);
}
private void prune(java.util.ArrayDeque&lt;Long&gt; q) {
    long cutoff = System.currentTimeMillis() - 1000L;
    while (!q.isEmpty() &amp;&amp; q.peekFirst() &lt; cutoff) q.pollFirst();
}

@Override
public int getLeftClickCps() {
    prune(leftClicks);
    return leftClicks.size();
}
@Override
public int getRightClickCps() {
    prune(rightClicks);
    return rightClicks.size();
}
</code></pre><p>Hook <code>onLeftClick()</code>/<code>onRightClick()</code> into your workspace's mouse-click event so the deque stays current.</p><hr><h2>Step 7 — Verify</h2><p>After completing the bridge and wiring events:</p><ol> <li>Launch the game. You should see the NeoCraft startup banner in the console/log.</li> <li>The HUD elements (FPS, Coordinates, etc.) should render if their modules are enabled.</li> <li>Press your ClickGUI keybind — the rounded ClickGUI should open with the category sidebar and module cards.</li> <li>Open Settings → Theme and switch presets; the live palette should update instantly.</li> <li>Create and switch profiles in Settings → Profiles; module/HUD state should isolate per profile.</li> <li>Exit and relaunch — all state should persist.</li> </ol><p>If anything throws <code>UnsupportedOperationException</code>, you missed uncommenting a method body in the bridge.</p><hr><h2>Adapter method reference</h2><p>The complete <code>MinecraftAdapter</code> interface (45 methods) is documented in <code>src/main/java/org/neocraft/api/MinecraftAdapter.java</code>. Here is the full list grouped by concern:</p><p><strong>Identity:</strong> <code>getClientBrand()</code>, <code>getMinecraftVersion()</code></p><p><strong>Timing:</strong> <code>currentTimeMillis()</code>, <code>getPartialTicks()</code>, <code>getFps()</code>, <code>getFrameDelta()</code></p><p><strong>Input:</strong> <code>isKeyDown(int)</code>, <code>isMouseLeftDown()</code>, <code>isMouseRightDown()</code>, <code>getMouseX()</code>, <code>getMouseY()</code>, <code>getScaledWidth()</code>, <code>getScaledHeight()</code></p><p><strong>Player/world:</strong> <code>isInGame()</code>, <code>isInWorld()</code>, <code>getPlayerX()</code>, <code>getPlayerY()</code>, <code>getPlayerZ()</code>, <code>getPing()</code>, <code>getLeftClickCps()</code>, <code>getRightClickCps()</code></p><p><strong>Gameplay:</strong> <code>getHealth()</code>, <code>getMaxHealth()</code>, <code>getFood()</code>, <code>getArmor()</code>, <code>getReach()</code>, <code>getGamma()</code>, <code>setGamma(float)</code>, <code>getFov()</code>, <code>setFov(float)</code>, <code>isSprinting()</code>, <code>setSprinting(boolean)</code>, <code>isSneaking()</code>, <code>getMoveSpeed()</code></p><p><strong>Rendering:</strong> <code>renderer()</code>, <code>fonts()</code>, <code>enableGuiMatrix()</code>, <code>disableGuiMatrix()</code></p><p><strong>Persistence:</strong> <code>config()</code>, <code>takeScreenshot()</code></p><p>The <code>StubAdapter</code> in <code>src/main/java/org/neocraft/adapter/StubAdapter.java</code> is a complete reference for expected return values and defaults, and is the easiest template to mirror when implementing <code>McpAdapter</code>.</p>
