<h1>NeoCraft</h1><p><strong>NeoCraft</strong> is a modern, original PvP client layer for <strong>Minecraft 1.12.2</strong> (Eaglercraft-compatible), built around a clean adapter architecture. The entire client — branding, theming, module framework, HUD, GUI, animations, profiles, notifications, and a built-in PvP resource pack — is written as portable, MIT-licensed Java that contains <strong>zero</strong> proprietary Minecraft code. It compiles and fully unit-tests in any standard JDK 17 environment, then drops into your own 1.12.2 workspace through a thin integration bridge.</p><blockquote> <p><em>"The future of PvP."</em></p> </blockquote><p>NeoCraft was inspired by the feature set of the Tuff Network's Eaglercraft PvP client (TuffClient) and the open-source TuffXPlus server plugin. It is, however, a <strong>completely separate and original</strong> codebase: no TuffClient binaries or decompiled code are included, and no TuffXPlus plugin code is distributed. Full attribution is preserved in the <a href="LICENSE">LICENSE</a> and on the in-game credits screen.</p><hr><h2>Table of Contents</h2><ul> <li><a href="#highlights">Highlights</a></li> <li><a href="#architecture">Architecture</a></li> <li><a href="#feature-catalogue">Feature Catalogue</a></li> <li><a href="#project-structure">Project Structure</a></li> <li><a href="#building">Building</a></li> <li><a href="#testing">Testing</a></li> <li><a href="#integrating-into-minecraft-1122">Integrating into Minecraft 1.12.2</a></li> <li><a href="#configuration--profiles">Configuration &amp; Profiles</a></li> <li><a href="#theming">Theming</a></li> <li><a href="#license">License</a></li> </ul><hr><h2>Highlights</h2><ul> <li><strong>Original, MIT-licensed source.</strong> The NeoCraft core never imports <code>net.minecraft.*</code>. All game interaction flows through a single <code>MinecraftAdapter</code> interface, so the project is clean-room and free of proprietary dependencies.</li> <li><strong>Compiles and tests anywhere.</strong> A self-contained <code>StubAdapter</code> lets the entire client build and run its 87-test suite in a plain JDK 17 sandbox — no Minecraft jars required.</li> <li><strong>One-file integration.</strong> Ship the core jar plus the MCP bridge (<code>McpAdapter</code>) into your Eaglercraft 1.12.2 workspace and call <code>NeoCraft.init(new McpAdapter())</code>. See <a href="INTEGRATION.md">INTEGRATION.md</a>.</li> <li><strong>Tuff-style PvP modules.</strong> Fullbright, Sprint, Zoom, Reach Display, Keystrokes, CPS, Coordinates, FPS, Ping, Armor HUD, Potion HUD, Clock, Session Stats, and a Notifications trigger — each with toggle, keybind, and configurable settings.</li> <li><strong>Modern GUI.</strong> Redesigned main menu with glass logo panel and glow, animated loading screen with progress bar and cycling messages, a polished ClickGUI with category sidebar, scroll, search, favorites, and expandable per-module settings, plus a tabbed settings screen and a credits screen.</li> <li><strong>Centralized theme system.</strong> Five built-in presets (NeoCraft Cyan, Amethyst, Emerald, Sunset, Crimson) with a live palette that swaps instantly. Custom presets can be installed at runtime.</li> <li><strong>Draggable HUD editor.</strong> Nine HUD elements in translucent rounded panels, freely repositioned and clamped, with layout persisted per profile.</li> <li><strong>Profiles, notifications, screenshots.</strong> Named switchable config namespaces, toast-style severity-colored notifications with auto-dismiss, and a screenshot manager with a 24-item recent list.</li> <li><strong>Deterministic and tested.</strong> Animations and auto-dismiss use an adapter-provided clock (not <code>System.currentTimeMillis()</code>), so every GUI animation and notification lifecycle is unit-testable.</li> <li><strong>Zero-warning build.</strong> The entire core compiles under <code>-Xlint:all -Werror</code> with no warnings.</li> </ul><hr><h2>Architecture</h2><p>NeoCraft follows an <strong>adapter pattern</strong> that cleanly separates portable logic from platform-specific wiring.</p><pre><code>        ┌─────────────────────────────────────────────┐
        │              NeoCraft core                   │
        │  (client, gui, hud, modules, theme, render,  │
        │   events, util) — no net.minecraft imports   │
        └──────────────────┬──────────────────────────┘
                           │ talks only through
                           ▼
                ┌──────────────────────┐
                │   MinecraftAdapter    │  (interface)
                │   GameRenderer        │
                │   FontRenderer        │
                │   ConfigStore         │
                └─────┬────────────┬────┘
                      │            │
          ┌───────────▼──┐    ┌────▼──────────────┐
          │  StubAdapter │    │   McpAdapter       │
          │  (sandbox)   │    │   (1.12.2 bridge)  │
          └──────────────┘    └────────────────────┘
</code></pre><p>The <code>org.neocraft.api</code> package defines four interfaces — <code>MinecraftAdapter</code>, <code>GameRenderer</code>, <code>FontRenderer</code>, and <code>ConfigStore</code> — that together describe everything NeoCraft needs from the host game. The <code>org.neocraft.adapter</code> package provides <code>StubAdapter</code> (and <code>StubRenderer</code>, <code>StubFont</code>, <code>StubConfig</code>) for sandbox compilation and testing. The <code>integration</code> source set provides the real <code>McpAdapter</code> family, which is compiled only inside the user's 1.12.2 workspace where MCP-mapped classes are available.</p><p>This means the core jar (<code>NeoCraft-1.0.0-core.jar</code>) is fully portable and testable, while the integration jar (<code>NeoCraft-1.0.0-integration.jar</code>) is the thin glue you drop into your mod.</p><hr><h2>Feature Catalogue</h2><h3>Modules (16)</h3><table class="e-rte-table"> <thead> <tr> <th>Module</th> <th>Category</th> <th>Description</th> </tr> </thead> <tbody><tr> <td>Fullbright</td> <td>Render</td> <td>Locks gamma to maximum for full night vision.</td> </tr> <tr> <td>Sprint</td> <td>Movement</td> <td>Force-sprints while moving forward.</td> </tr> <tr> <td>Zoom</td> <td>Render</td> <td>Reduces FOV for a scoped view; gradle build
</code></pre><p>This compiles the core against the stub, runs the full test suite, and produces:</p><ul> <li><code>build/libs/NeoCraft-1.0.0-core.jar</code> — the portable, dependency-free core.</li> </ul><h3>Build the integration jar</h3><pre><code class="language-bash">gradle integrationJar
</code></pre><p>This compiles the <code>/integration</code> source set (which depends on the core's output) and produces:</p><ul> <li><code>build/libs/NeoCraft-1.0.0-integration.jar</code> — the MCP bridge.</li> </ul><blockquote> <p>The integration jar compiles in this sandbox because the bridge methods are currently stubs that throw <code>UnsupportedOperationException</code>. To make them functional, uncomment the MCP-mapped implementation hints inside each <code>Mcp*.java</code> file and compile inside your 1.12.2 workspace. See <a href="INTEGRATION.md">INTEGRATION.md</a>.</p> </blockquote><h3>Compile-only checks</h3><pre><code class="language-bash">gradle compileJava            # core only

import org.neocraft.adapter.McpAdapter;

NeoCraft.init(new McpAdapter());
</code></pre><ol start="5"> <li>Route your render tick, input, and key events into <code>NeoCraft.get().events()</code>.</li> </ol><p>For the complete, annotated walkthrough — including event routing, CPS counter wiring, and HUD/ClickGUI open hooks — see <strong><a href="INTEGRATION.md">INTEGRATION.md</a></strong>.</p><hr><h2>Configuration &amp; Profiles</h2><p>NeoCraft persists all state through the <code>ConfigStore</code> interface. In the sandbox this is an in-memory stub; in 1.12.2 it maps to a <code>Properties</code> file at <code>.minecraft/neocraft/config.properties</code>.</p><p><strong>Profiles</strong> are isolated namespaces (<code>profile.&lt;name&gt;.&lt;key&gt;</code>). You can create, switch to, and delete profiles from the Settings screen. The default profile (<code>default</code>) cannot be deleted. Deleting the active profile reverts to <code>default</code>. The active store is cached per profile name for performance and invalidated on switch.</p><p>Persisted state includes: module enabled/disabled flags, module settings, keybinds, favorites, HUD element positions, the active theme preset, the active profile, resource pack toggle, and general settings (notifications, screenshots, FPS/coordinates HUD).</p><hr><h2>Theming</h2><p>NeoCraft ships <strong>five built-in theme presets</strong>, all swappable live from the Settings → Theme tab:</p><table class="e-rte-table"> <thead> <tr> <th>Preset</th> <th>Primary</th> <th>Secondary</th> <th>Vibe</th> </tr> </thead> <tbody><tr> <td>NeoCraft (Cyan)</td> <td><code>#00E5FF</code></td> <td><code>#8B5CF6</code></td> <td>Default — cool cyan/purple</td> </tr> <tr> <td>Amethyst</td> <td><code>#8B5CF6</code></td> <td><code>#00E5FF</code></td> <td>Purple-led</td> </tr> <tr> <td>Emerald</td> <td><code>#38D39F</code></td> <td><code>#00E5FF</code></td> <td>Green-led</td> </tr> <tr> <td>Sunset</td> <td><code>#F5B041</code></td> <td><code>#EF4444</code></td> <td>Warm amber/red</td> </tr> <tr> <td>Crimson</td> <td><code>#EF4444</code></td> <td><code>#8B5CF6</code></td> <td>Red-led</td> </tr> </tbody></table><p>The shared background (<code>#0E1016</code>), panel (<code>#181C24</code>), and text (<code>#F5F7FA</code>) tones keep every preset coherent.
